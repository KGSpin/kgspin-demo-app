What do these modes do?
Chunk-anchored chunk_first
Retrieves chunks via dense (BM25 + cosine RRF), then for each chunk attaches its in-chunk subgraph — entities whose evidence falls in that chunk plus their N-hop neighborhood (default N=3, configurable via graph_rag.n_hops_default). Each chunk row in the prompt carries its own subgraph. Recommended default: chunks anchor the answer, graph adds structure.
Graph-anchored graph_first
Retrieves graph items (nodes/edges) via BM25 + cosine RRF over canonical entity names + edge predicates. For each match, pulls in the source chunk(s) containing the evidence. Graph drives retrieval, chunks are evidence. Best when the question maps to canonical entities or relationships.
Parallel parallel
Runs dense and graph searches independently — no joining at retrieval time. Both result lists land side-by-side in the prompt as separate [DENSE RETRIEVAL] and [GRAPH RETRIEVAL] sections. Lets the LLM see both retrieval signals as independent columns of context.
