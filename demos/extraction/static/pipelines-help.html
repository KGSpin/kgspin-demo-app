KGSpin Pipelines — Help

Canonical descriptions of the five extraction pipelines exposed in the compare-UI. Source of truth: docs/architecture/product_naming_proposal.md and kgspin-core/docs/architecture/pipelines.md.

Jump to: Rapid Discovery Deep Discovery Signal Fan-out Agentic Flash Agentic Analyst

Rapid Discovery 0 tokens

Capability: Discovery

Entity-first, zero-token linguistic baseline. Uses spaCy NER plus the Greedy Harvester and disables all neural relation heads. The fastest actor-inventory baseline — good for quickly surfacing who/what a document is about without paying for neural inference.

Strategy: discovery_rapid   Pipeline YAML: discovery-rapid.yaml

Deep Discovery 0 tokens

Capability: Discovery

Entity-first, zero-token neural-hybrid. GLiNER per-chunk entity detection plus enabled neural relation heads (GLiREL / NLI / pairwise). The "Restored Greedy Neural" path — high-density actor inventory with validated links. This is the canonical discovery-density baseline and is the default for slot 2.

Strategy: discovery_deep   Pipeline YAML: discovery-deep.yaml

Note: Post-INIT-006 Amendment 002 (2026-04-14), this path is pure zero-token — any supplied chat backend is silently ignored.

Signal Fan-out 0 tokens

Capability: Fan-out

Relation-first, zero-token. Identifies relationship anchors first, then "fans out" to bind the relevant actors around each anchor. The complement to Discovery strategies — where Discovery asks "who is in this document", Fan-out asks "what are the events/relationships in this document, and who participates". Default for slot 1.

Strategy: fan_out   Pipeline YAML: fan-out.yaml

Agentic Flash LLM

Capability: Agentic

Single-prompt monolithic LLM extraction. Lower latency and a simpler prompt than Agentic Analyst, and uses the generic NER types (ORGANIZATION, PERSON). Good for a baseline comparison of "what does a raw LLM pull out" against the zero-token KGSpin pipelines.

Strategy: agentic_flash   Pipeline YAML: agentic-flash.yaml

Agentic Analyst LLM

Capability: Agentic

Schema-aware chunked LLM extraction with entity-map carryover across chunks. Highest fidelity of the agentic path — uses the bundle's richer types (e.g., COMPANY, CORPORATE_LEADER) and maintains cross-chunk context. Default for slot 3.

Strategy: agentic_analyst   Pipeline YAML: agentic-analyst.yaml

Setting EDGAR_IDENTITY

Configuration

Live SEC EDGAR fetch (required for any ticker other than the offline demo_mock fixtures, e.g. MSFT / PFE / AAPL) requires an EDGAR_IDENTITY environment variable. SEC compliance rules mandate that all EDGAR API consumers identify themselves — an email address is sufficient — and the SEC EDGAR lander enforces this at construction time.

One-line setup:

export EDGAR_IDENTITY='you@example.com'

Then re-run the demo:

./scripts/start-demo.sh

Once set, every ticker in the dropdown (and any free-text ticker) will work against live SEC data. The first fetch per ticker is slow (one live EDGAR HTTP round-trip); subsequent fetches hit the local cache under ~/.kgspin/corpus/.

Offline mode: if EDGAR_IDENTITY is unset, only tickers with fixture files at tests/fixtures/corpus/{ticker}.html will work. The default fixture is JNJ; other tickers will surface an actionable error in the slot.
